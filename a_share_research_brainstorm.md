# 中国 A 股中低频投资策略与风控研究系统（个人版）脑暴方案

## 1. 目标再定义（与你的约束对齐）
- **第一目标**：建立“可持续迭代”的研究与验证流水线，而不是先做交易执行平台。
- **资金规模**：50 万～500 万人民币，核心矛盾是**回撤控制、容量约束、换手约束、可执行性**。
- **市场与频率**：A 股日线/周线中低频；不做高频，不依赖盘口。
- **风格偏好**：高夏普、低回撤、可解释、可复现、可版本化。

> 推荐路线：先做“研究 OS”，再做“轻量实盘适配”。

---

## 2. 系统架构（最小可用但可扩展）

### 2.1 分层
1. **Data Layer（数据层）**
   - 标的池、行情、复权因子、财务、行业、指数成分、ST/停牌/涨跌停状态、交易日历。
   - 关键要求：**Point-in-Time（时点一致）**，避免未来函数。

2. **Research Layer（研究层）**
   - 因子研究、择时研究、横截面选股、组合构建、交易规则、成本冲击建模。
   - 支持参数扫描与 Walk-Forward（滚动训练/验证）。

3. **Risk Layer（风控层）**
   - 组合风控（波动目标、回撤阈值、仓位上限、行业暴露、单票集中度）。
   - 交易风控（开仓冷却、止损/止盈、黑名单、极端波动熔断）。

4. **Evaluation Layer（评估层）**
   - 分 regime（牛/熊/震荡）评估、分年份评估、分市值风格评估。
   - 指标：Sharpe、Sortino、Calmar、maxDD、回撤恢复期、换手、容量、胜率/盈亏比。

5. **Versioning Layer（版本层）**
   - 数据版本、代码版本、实验版本、参数版本、结果版本。
   - 每个实验可追溯：数据快照 hash + commit id + config。

6. **Execution Adapter（执行适配层，可后置）**
   - 仅保留“信号 -> 目标仓位 -> 委托列表”的薄适配层，不绑定重平台。

---

## 3. 技术选型建议（开源框架组合，不追求“一把梭”）

### 3.1 推荐组合 A（更贴近 A 股与个人研究）
- **核心研究/回测引擎**：RQAlpha 或 Backtrader（事件驱动，规则与撮合可控）
- **数据接口**：AKShare（快速接入公开数据） + 本地 Parquet 数据仓
- **因子/ML研究**：Qlib（用于因子工程、模型比较、实验管理思想）
- **工程化**：
  - 实验管理：MLflow 或 lightweight SQLite + YAML registry
  - 编排：Makefile + Python CLI（Typer）
  - 存储：Parquet + DuckDB

适合你这种“中低频 + 可解释 + 成本敏感 + 可复现”的路径。

### 3.2 推荐组合 B（强调向量化高速迭代）
- **策略原型**：vectorbt（大规模参数扫描）
- **精细事件回放**：Backtrader / RQAlpha（二次确认真实交易约束）

做法：先向量化粗筛，再事件驱动精筛，显著节省研究时间。

## 4. 数据与“反未来函数”治理（成败关键）

1. **必须落地本地数据仓**（按交易日增量更新）
2. **必须保存成分股历史**（指数调入调出）
3. **必须使用 PIT 财务口径**（公告日可见）
4. **必须保留交易状态约束**：停牌/ST/涨跌停不可成交
5. **必须有成本模型**：
   - 双边手续费 + 印花税（卖出）
   - 滑点：按成交额占日成交额比例做分段惩罚

> 你资金规模下，成本和冲击对周频/日频策略影响非常实在。

---

## 5. 策略研究框架（建议优先做三类）

### 5.1 横截面多因子选股（核心）
- 风格：质量 + 价值 + 动量 + 低波的稳健组合
- 操作：周频调仓、等权或风险平价
- 风控：行业中性约束 + 单票上限 + 换手上限

### 5.2 市场状态择时（风险开关）
- 用指数趋势 + 波动状态 + 宽基 breadth（上涨家数占比）
- 输出：仓位上限因子（如 30% / 60% / 100% 档位）

### 5.3 回撤控制策略（组合层）
- 波动率目标（target vol）
- 动态降杠杆/降仓位规则
- 连续亏损冷却期（N 周）

---

## 6. 风控研究体系（建议独立成“策略之上策略”）

将风控参数化并单独研究，不要写死在策略内部：

- `max_single_weight`
- `max_industry_weight`
- `max_turnover`
- `max_drawdown_trigger`
- `vol_target`
- `stop_loss_mode`
- `cooldown_days`

这样你就可以做：
- 同一 alpha + 多套风控模板对比
- 同一风控模板 + 多个 alpha 对比

最终得到的是“策略 x 风控”矩阵，而不是单一净值曲线。

---

## 7. 研究验证方法（避免过拟合）

1. **时间切分**：
   - 训练期 / 验证期 / 测试期（严格时间前后）
2. **Walk-Forward**：
   - 每年或每季度滚动重估参数
3. **多市场状态验证**：
   - 上涨期、下跌期、震荡期分别看表现
4. **压力测试**：
   - 成本上调 1.5x / 2x
   - 滑点扩大
   - 延迟一个 bar 执行
5. **稳定性筛选而非收益最大化**：
   - 参数邻域稳健（小改参数不崩）
   - 分段业绩一致性

## 9. 12 周落地路线图（实操版）

### Phase 1（第 1-2 周）：数据地基
- 建交易日历、复权行情、成分股历史、交易状态表
- 统一写入 Parquet + DuckDB

### Phase 2（第 3-5 周）：回测引擎与成本模型
- 跑通日/周频回测
- 加入 A 股交易制度约束 + 成本模型

### Phase 3（第 6-8 周）：策略与风控模板
- 完成 1 套多因子选股 + 1 套择时 + 2 套风控模板
- 形成可复用策略接口

### Phase 4（第 9-10 周）：验证与压力测试
- Walk-forward + 参数稳定性 + 成本敏感性

### Phase 5（第 11-12 周）：版本化与报告自动化
- 一键生成实验报告（指标、净值、回撤、持仓、暴露）
- 形成“可投产候选池”

---

## 10. 你这个规模下最容易踩的坑

1. 只看收益不看回撤恢复时间
2. 忽略交易状态导致“纸面可交易、实盘不可交易”
3. 未做指数成分历史还原，样本污染
4. 参数搜索过深，策略失真
5. 把风控逻辑和 alpha 搅在一起，无法复用

---

## 11. 最终建议（简版）

- **先做研究系统，不先做交易系统**。
- **先做可复现，不先做复杂 AI**。
- **先做稳健风控，不先做收益极值**。
- 技术栈推荐：
  - 数据：AKShare + Parquet + DuckDB
  - 回测：RQAlpha/Backtrader
  - 研究扩展：Qlib（因子与实验框架思想）
  - 迭代提速：vectorbt 做参数粗筛
